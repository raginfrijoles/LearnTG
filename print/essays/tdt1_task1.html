Overview and Expectations

        There are a variety of methods for creating a website. Platforms like Wix and Weebly allow for easy template creation and encourage the user to focus on presentation and design. However, these platforms tend to limit source code manipulation and keep the user within the “skeleton” of a template. Although this is convenient for creating clean and functional websites quickly, creating a website from scratch gives the creator the freedom to create tailored interactive experiences.

This guide will cover the following steps required for creating a website from scratch: tools, file structure and file creation, frameworks, HTML/CSS setup, host selection, and FTP upload. Be aware that basic knowledge of HTML/CSS markup and Javascript will be helpful. For any questions regarding these technologies, please reference the website W3Schools for basic HTML, CSS, and Javascript rules and implementations (HTML, 2016).

Creating a Website

Tools

        There are a multitude of tools available for creating a website from the ground up, because the list is nearly endless, I will be covering the three that I use most often: Sublime Text, Adobe Dreamweaver, and FileZilla. Sublime Text is a simple text editor that streamlines file structures and provides helpful markup (visual cues that hint to potential errors) when writing HTML and Javascript. Adobe Dreamweaver is a similar text editor but more robust and provides mobile preview features and visual markup. If you consider yourself a beginner, hold off on using Dreamweaver since its interface can be daunting and counterproductive. Lastly, FileZilla is a free FTP client that we will use to upload our website to a web server.  Although FileZilla is the only tool that is required, (you could theoretically create everything within Notepad or TextEdit) you will be spending a lot of time navigating file structures and writing text so try to use a tool that will speed up your workflow.

File Structure and File Creation

Before any HTML, CSS, or Javascript is written, it is important to create a file structure that is logical and easy to navigate. Start off by creating a root folder where everything in your website is going to live. For this guide I will call the folder “root”. You can call this folder anything, but make sure it is something that makes sense. Within this root folder, create three folders called “img”, “css”, and “js”. These folders will house your images, “.html” files, “.css” files, and “.js” files.

Now that we have all the folders that we need, we must create the skeleton files that we will write our code in later. In the root folder, create a file named “index.html”. This can be done in Sublime Text by navigating to File > New File > Save As... > and typing “index.html”. Your “index.html” page will be your landing page for your website so try not to name this file anything weird since the web server will look for a “index” file in your website’s root folder. Create files for each additional page that you would like to have as well. In this example, I created files named “multimedia.html”, “print.html”, and “graphics.html”. Next, create a “main.css” file in the “css” folder and a “main.js” file in the “js” folder using the same commands. Lastly, create “.css” and “.js” files for each of your pages in the “css” and “js” folders. Take note that these names are meant to help you identify files and you will be referencing them often while you code, so keep the names short and sweet. The resulting file structure should look something like the one below.

Frameworks

        Now that we have a workable file structure, theoretically, we can begin writing out our HTML and CSS. This is especially time-consuming so we will speed up our workflow by implementing “frameworks” that contain sets of  predefined Javascript and CSS “rules” that quicken site creation. The two frameworks I recommend downloading are Bootstrap and jQuery. These frameworks provide a simpler syntax like jQuery’s “.hide();” to speed up Javascript functionality and Bootstrap’s predefined classes like “.row” and “.col” to speed up CSS styling (CSS, 2016; jQuery API Documentation, 2016). When you download these frameworks, they will provide you with “.js” and “.css” files. Place these files in your “css” and “js” folders within your root folder. We will link these files up to our pages later.

HTML and CSS Setup

        Now we can begin writing our HTML and CSS. Open your “index.html” file and write in the following code:

index.html

<!DOCTYPE html>

<html>

<head>

<meta charset="UTF-8" name="viewport" content="width=device-width, initial-scale=1">

<link rel="stylesheet" type="text/css" href="css/bootstrap.min.css">

<link rel="stylesheet" type="text/css" href="css/main.css">

<script src = "js/jquery-2.2.4.min.js"></script>

<script src = "js/bootstrap.min.js"></script>

<script src = "js/main.js"></script>

<title>Page Title</title>

</head>

<body>

</body>

</html>

This will serve as the skeleton for all of your webpages. To summarize, we are telling the browser that we are using an HTML document type and telling it which frameworks and files to use in our file structure. Each “<script></script>” and “<link>” tag tells the browser where to find our framework or custom file. For example, the “<script src = "js/jquery-2.2.4.min.js"></script> ” tag looks for the jQuery “.js” file that we downloaded earlier in our “js” folder. For specifics on what exactly each tag does, reference W3Schools (HTML, 2016).

        Now that we have our document setup, we can start customizing our webpage. For the majority of our customization, we will be using the set of CSS rules provided by Bootstrap. All of this work will be done within the “<body>” and “</body>” tag. Enter in the following code to create a “jumbotron” presentation piece.

index.html

<!DOCTYPE html>

<html>

<head>

<meta charset="UTF-8" name="viewport" content="width=device-width, initial-scale=1">

<link rel="stylesheet" type="text/css" href="css/bootstrap.min.css">

<link rel="stylesheet" type="text/css" href="css/main.css">

<script src = "js/jquery-2.2.4.min.js"></script>

<script src = "js/bootstrap.min.js"></script>

<script src = "js/main.js"></script>

<title>Page Title</title>

</head>

<body>

<div class="jumbotron text-center">

  <h1>Your Page Heading</h1>

  <h3>Your Page Sub-heading</h3>

</div>

</body>

</html>

This will create a nice area to put your page heading. The default styling can be changed by adding some CSS rules to your “main.css” file. In my case, I altered the heading fonts and added an image to the back of the “jumbotron” by entering the following code in “main.css”.

main.css

.jumbotron{

        background-size:cover;

        background-image:url(../img/jumbotron.jpg);

        box-shadow:0px 2px 5px #bbb;

}

.jumbotron h1, .jumbotron h3{

        color:#fff;

        text-shadow: 0 1px 0  #cccccc,

               0 2px 0  #b3b3b3,

               0 3px 0 #666666,

               0 4px 0 #666666,

               0 0 5px rgba(0,0,0,.10),

               0 1px 3px rgba(0,0,0,.3),

               0 3px 5px rgba(0,0,0,.2),

               0 5px 10px rgba(0,0,0,.24),

               0 10px 10px rgba(0,0,0,.2),

               0 20px 20px rgba(0,0,0,.14);

}

 

If you are unfamiliar with how CSS works, just know that it is used to change how each element is displayed on your page. In the above code, I selected the “.jumbotron” element and gave it a background image and drop-shadow. It is important to note that since your “main.css” file is housed in your “css” folder, you need to use the “../” identifier when searching for images in your “img” folder. This will tell your css file that the image is in your “root/img” folder. Otherwise, if you typed something like “img/image.jpg”, your “main.css” file would look for your image in “css/img/image.jpg” which is a directory we did not create. For more styling rules, reference W3Schools CSS section (CSS, 2016).

        Now that we have some basic styling rules and a page heading, we need to create a navigation bar so that our visitors can navigate to other pages. For this, we will use Bootstrap’s button group preset. Bootstrap has a ton of different navigation options, but I think the button group is the simplest to implement for basic web pages. To create a button group, enter the following code under the “.jumbotron” element we created earlier in “index.html”.

index.html

<!DOCTYPE html>

<html>

<head>

<meta charset="UTF-8" name="viewport" content="width=device-width, initial-scale=1">

<link rel="stylesheet" type="text/css" href="css/bootstrap.min.css">

<link rel="stylesheet" type="text/css" href="css/main.css">

<script src = "js/jquery-2.2.4.min.js"></script>

<script src = "js/bootstrap.min.js"></script>

<script src = "js/main.js"></script>

<title>Page Title</title>

</head>

<body>

<div class="jumbotron text-center">

  <h1>Your Page Heading</h1>

  <h3>Your Page Sub-heading</h3>

</div>

<div class = "row">

        <div class = "col-xs-12 text-center">

                <div class="btn-group">

                         <a href="index.html" class="btn btn-default">Home</a>

                          <a href="graphics.html" class="btn btn-default">Graphics</a>

                          <a href="multimedia.html" class="btn btn-default">Multimedia</a>

                          <a href="print.html" class="btn btn-default">Print</a>

                </div>

        </div>

</div>

</body>

</html>

You should notice that we added three “<div></div>” tags in order to create this button group. The first “<div class = “row”></div>” tag has the class “row”. This is a Bootstrap rule that splits the user’s screen into twelve distinct columns so that we can manage how the content is shown on various devices. Within this “row” we add a “<div class = “col-xs-12”></div> tag that tells the browser we want this section to display across all twelve columns on “xs” devices (“xs” stands for extra small devices like an iPhone). I recommend reading Bootstrap’s documentation regarding “.col” rules in order to better manipulate your content (CSS, 2016).  Lastly within the third “<div class = “btn-group></div>” tag, we created four “<a></a>” tags that utilize the “href” attribute. This will serve as your bread and butter for navigation. The “href” attribute creates a link and will navigate to the destination you specify. Within the “href” attribute type in the location of the page you would like to link. This can be a page within your directory like “multimedia.html”, or it can be an outside source like “http://google.com”.

        Now that we have our heading and navigation set up, we can start adding the actual content. For all of your content, you will be utilizing “row”, “col”, “h1”, “img” and “p” elements. Below you can find an example of how to begin displaying your content.

index.html

<!DOCTYPE html>

<html>

<head>

<meta charset="UTF-8" name="viewport" content="width=device-width, initial-scale=1">

<link rel="stylesheet" type="text/css" href="css/bootstrap.min.css">

<link rel="stylesheet" type="text/css" href="css/main.css">

<script src = "js/jquery-2.2.4.min.js"></script>

<script src = "js/bootstrap.min.js"></script>

<script src = "js/main.js"></script>

<title>Page Title</title>

</head>

<body>

<div class="jumbotron text-center">

  <h1>Your Page Heading</h1>

  <h3>Your Page Sub-heading</h3>

</div>

<div class = "row">

        <div class = "col-xs-12 text-center">

                <div class="btn-group">

                         <a href="index.html" class="btn btn-default">Home</a>

                          <a href="graphics.html" class="btn btn-default">Graphics</a>

                          <a href="multimedia.html" class="btn btn-default">Multimedia</a>

                          <a href="print.html" class="btn btn-default">Print</a>

                </div>

        </div>

</div>

<div class = "row">

        <div class = "col-xs-12 col-sm-8 col-sm-offset-2 col-md-6 col-md-offset-3">

                <h1 class = "text-center">Welcome</h1>

                <p>This is your introduction.</p>

                <h4>Sample Content</h4>

                <img class = "img-responsive" src = "http://tinyurl.com/jhy3757"/>

                <p class = "indent">Description of content.</p>

        </div>

</div>

</body>

</html>

Similar to the button group, we created a “row” and applied a set of “col” rules to our content section. The “col” rules specify that the content will display across all twelve columns when a device is extra-small (“col-xs-12”). If the device is considered a small device or “sm”, the content will take up only eight columns and move over two columns (“col-sm-8 col-sm-offset-2”). If the device is medium sized or “md”, the content takes up six columns and moves over by three columns (“col-md-6 col-md-offset-3”). This “col” logic is essential for manipulating what your content will look like across all devices (CSS, 2016). Within the column, an image was added using the “<img/>” tag and a Bootstrap rule “img-responsive” was applied to make sure that the image always fits within the column. You can test how the image reacts by adjusting the size of your browser window and observing the image shrink and grow. If you want to add additional media, please reference Bootstrap’s component page for video web embed, and W3Schools sections on video and audio (Components, 2016; HTML, 2016).

Now that we have a basic web page, we can begin adding interactivity by utilizing Javascript and jQuery. For this example, I will be creating a button that hides and shows content. Enter the following code in “index.html” within your content columns.

index.html

<div class = "row">

        <div class = "col-xs-12 col-sm-8 col-sm-offset-2 col-md-6 col-md-offset-3">

                <h1 class = "text-center">Welcome</h1>

                <p>This is your introduction.</p>

                <h4>Sample Content</h4>

                <img class = "img-responsive" src = "http://tinyurl.com/jhy3757"/>

                <p class = "indent">Description of content.</p>

                <h3>Sample Question: Does this work?</h3>

                <p>Does this work?</p>

                <p id = "answer">Yes. It works!</p>

                <button id = "mybutton" class = "btn btn-default">Toggle</button>

        </div>

</div>

This code displays the content we want to interact with, Javascript and jQuery will do the heavy lifting. Take note of the “id” attribute on the button and paragraph elements. These attributes are specific only to those elements and will be used to target these elements with jQuery. Now we need to open “main.js” and write some code using jQuery’s framework. Remember that “main.js” will be linked within all of your pages, so if you want specific interactivity on one page, create or edit a separate “.js” file like “print.js” and link it to the page you want to customize. Enter the following code in “main.js”.

main.js

$(document).ready(function(){

        $("#answer").hide();

        $("#mybutton").click(function(){

                $("#answer").toggle();

        });

});

This code hides the “answer” element that is created when  the document is “ready”. It then toggles our answer’s visibility when the “mybutton” element is clicked. Although jQuery uses a specific syntax, this is essentially a bunch of Javascript commands shortened into a few lines thanks to the jQuery framework. For more information, visit jQuery’s API documentation page to browse different options for making your website more engaging (jQuery API Documentation, 2016).

Host Selection

        Once your website is built and ready to go, you still need to host your content on a web server for the world to see. There are numerous ways you can do this, but the most common option is to purchase services from a hosting company. For my website, I used the hosting company iPage which costs around $12.00 a year. These companies will help you choose a domain name and provide you with server space to upload your website. There is no hard and fast rule to which hosting company you should use, but make sure you are aware of how each hosting company handles uploads.

Uploading via FTP

        The most common remote uploading method is the File Transfer Protocol or “FTP”. For this example, we will use the FileZilla client to connect to a web server and upload all the files and folders we created. Before you open FileZilla, make sure you know your site’s FTP address (specified by your web host, something like: ftp.something.com) and your username and password. Each company is different, but their process for obtaining these parameters are easy to find from a quick Google search.

Once you have your server address, username, and password, open FileZilla. Type in your server address under “host” and then your username and password. For the port number, use port number 21 unless directed by your web host. After connecting to your site’s server, you should see two split panes. On the left, are all the files on your local computer (like your website). On the right, are all the files that are currently on your web server. Depending on your web host, you should see a “root” or “site” folder on your web server. We want to put everything inside our local “root” folder into the “root” folder on our web server. This is as easy as navigating to the “root” folder on our local computer, selecting all the files inside of this folder and dragging these files inside the root folder on your web server. If done correctly, you should see a small progress bar at the bottom of FileZilla as your files upload to your server. You should now be able to navigate to the domain that you purchased by typing in your domain in a browser window.
